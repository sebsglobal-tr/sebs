Web Uygulama Güvenliği

Web uygulamalarında kimlik doğrulama, oturum yönetimi, erişim kontrolü, input validation, güvenli API kullanımı, loglama ve temel zafiyet sınıflarını savunma odaklı ele alan orta seviye güvenlik eğitimi.

Program özeti

Bu eğitim, temel web geliştirme ve siber güvenlik bilgisine sahip kullanıcıların uygulamaları güvenlik bakış açısıyla değerlendirmesini hedefler. HTTP yapısı, kimlik ve oturum riskleri, erişim kontrolü, doğrulama, güvenli hata yönetimi, API güvenliği, güvenli SDLC ve kanıt temelli raporlama bir bütün olarak ele alınır.

Güvenlik ve etik sınır: İçerik saldırı, istismar, bypass, yetkisiz erişim veya exploit zinciri öğretmez. Örnekler yerel lab, eğitim ortamı, kurumsal yetkili kapsam veya uydurma senaryolarla sınırlıdır.

Öğrenme çıktıları

  • Web uygulama mimarisini güvenlik bakış açısıyla açıklayabilme
  • HTTP, istek, yanıt, başlık, çerez ve oturum kavramlarını güvenlik açısından yorumlayabilme
  • Kimlik doğrulama (authentication) ve yetkilendirme (authorization) farkını ayırt edebilme
  • Kırık erişim kontrolü risklerini savunma perspektifiyle değerlendirebilme
  • Input validation, çıktı kodlama ve sunucu tarafı kontrol mantığını anlayabilme
  • SQL Injection, XSS, CSRF, SSRF gibi sınıfları saldırı öğretmeden; mekanizma, etki, sinyal ve savunma kontrolü üzerinden kavrayabilme
  • API güvenliğinde token, rate limit, yetkilendirme ve veri sızıntısı risklerini değerlendirebilme
  • Güvenli hata yönetimi, loglama ve izleme prensiplerini uygulama düzeyinde anlayabilme
  • Bulguları Bulgu → Etki → Öneri → Kanıt formatında raporlayabilme
  • Güvenli yazılım geliştirme yaşam döngüsü içinde temel güvenlik kontrollerini konumlandırabilme

Kimler için

  • Web uygulama güvenliği öğrenmek isteyen öğrenciler
  • AppSec, Blue Team, SOC veya güvenli yazılım geliştirme alanına yönelmek isteyen kullanıcılar
  • Web geliştiriciliğini güvenlik bakış açısıyla güçlendirmek isteyenler
  • API güvenliği ve güvenli kodlama farkındalığı kazanmak isteyenler
  • SEBS kariyer yolunda temel web ve temel güvenlik modüllerini tamamlayan kullanıcılar

Ön koşul

Temel web geliştirme bilgisi (HTML/CSS/JavaScript), HTTP mantığına başlangıç düzeyinde aşinalık, temel siber güvenlik ve temel network bilgisi.

Kimler için uygun değil

  • Web geliştirme temellerini hiç bilmeyen kullanıcılar
  • Yetkisiz test, exploit veya bypass pratiği arayan kullanıcılar
  • Savunma ve güvenli geliştirme yerine saldırı odaklı içerik arayan kullanıcılar

Müfredat

On bir ana ünite; alt başlıklar savunma ve doğrulama odaklıdır. Zafiyet anlatımlarında ad → anlam → oluşum → etki → sinyaller → önleme → doğrulama → rapor kanıtı çerçevesi kullanılır; exploit zinciri veya yetkisiz test anlatılmaz.
